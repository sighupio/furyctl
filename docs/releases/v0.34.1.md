# furyctl release v0.34.1

Welcome to the latest release of `furyctl` maintained by SIGHUP by ReeVo team.

## New features 🌟

- Added support for SIGHUP Distribution `v1.33.2` and `v1.32.2` in the compatibility matrix for all three providers (`EKSCluster`, `KFDDistribution`, `OnPremises`).
- Added the following upgrade paths:
  - Intra-patch: `1.33.1 → 1.33.2` and `1.32.1 → 1.32.2` (includes Kubernetes patch upgrade to `1.33.7` and `1.32.11` respectively on the `OnPremises` provider, and the Terraform state backup to S3 required for the transition to OpenTofu on `EKSCluster`).
  - Cross-minor: `1.32.2 → 1.33.2` (includes Kubernetes minor upgrade `1.32.11 → 1.33.7` on `OnPremises`) and `1.33.2 → 1.34.0` (includes `1.33.7 → 1.34.4` on `OnPremises`). Both paths keep module versions consistent to avoid module downgrades.

## Bug fixes 🐞

- [[#649](https://github.com/sighupio/furyctl/pull/649)] Validate that no positional arguments are passed to commands that do not support them.
- [[#650](https://github.com/sighupio/furyctl/pull/650)] Fallback to AMD64 binary for kustomize 3.10.0 (used in SD v1.31.0, for example) because ARM binary is not available.

## Breaking Changes 💔

None.
